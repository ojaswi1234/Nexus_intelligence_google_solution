# Nexus Supply Chain Intelligence 🌐📦

**Nexus Supply Chain Intelligence** is an AI-powered predictive monitoring platform that transforms raw logistics data into actionable intelligence. By combining real-time tracking with advanced AI reasoning, Nexus helps logistics managers stay ahead of global disruptions.

## ✨ Key Features

- **Global Transit Map**: Interactive D3-powered visualization of real-time shipment paths and active disruption zones.
- **AI Risk Intelligence**: Automated executive summaries generated by **Gemini 3 Flash**, identifying critical threats to the network.
- **AI Command Center**: A conversational assistant for querying shipment status, analyzing disruption impact, and generating optimized route waypoints.
- **Live Operational Feed**: Real-time Firestore-backed updates on shipment status, risk levels (Low/Medium/High), and ETA adjustments.
- **Admin Seeding Tool**: Built-in utility for populating the network with operational data for testing and demonstrations.

## 🛠️ Tech Stack

- **Frontend**: React 19, Vite, Tailwind CSS (v4), Motion (Framer Motion)
- **Backend**: Firebase Firestore (Real-time DB), Firebase Authentication (Google Login)
- **AI/ML**: Google Gemini API (`@google/genai`)
- **Data Visualization**: D3.js (Geographic mapping & transit paths)
- **Icons**: Lucide React

## 🚀 Getting Started

### Prerequisites

- Node.js (v18+)
- A Google Cloud Project with the **Gemini API** enabled.
- A Firebase Project with **Firestore** and **Authentication** (Google) enabled.

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/ojaswi1234/nexus-intelligence-google-solution.git
   cd nexus-intelligence-google-solution
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Configure Environment Variables:**
   Create a `.env` file in the root directory:
   ```env
   GEMINI_API_KEY=your_gemini_api_key_here
   ```

4. **Firebase Configuration:**
   Place your `firebase-applet-config.json` in the root directory:
   ```json
   {
     "apiKey": "...",
     "authDomain": "nexus-intelligence-google-solution.vercel.app",
     "projectId": "...",
     "appId": "...",
     "firestoreDatabaseId": "..."
   }
   ```

5. **Run the development server:**
   ```bash
   npm run dev
   ```

## 🔒 Security Rules

This project includes a `firestore.rules` file designed with **Default Deny** principles, validating all incoming data types, string lengths, and enforcing role-based access control (RBAC).
